# Kanban Skill

异步任务管理系统，配合 `~/todo/` 目录使用。

## 触发条件

用户说以下任一指令时触发：
- "添加 todo"
- "加个任务"
- "task add"
- "帮我记个任务"
- 或任何表达"添加任务到待办"的意图

## 执行规则

### 添加任务（只执行这个，不要执行任务本身！）

1. 解析任务信息：
   - 优先级：`[P0-P4]`，默认 P2
   - 项目名：`[项目名]`，可选
   - 任务描述：具体做什么

2. 写入 `~/todo/inbox.md`：
```
[P优先级] [项目名] 任务描述
```

3. **重要**：只写 inbox，**不要立即执行任务**！等 cron 触发或用户主动要求检查时再处理。

4. 回复用户确认已添加。

### 查看进度

用户说"查看 todo"/"任务进度"/"还有哪些任务"时：
- 读取 `~/todo/active.md` 和 `~/todo/backlog.md`
- 汇总当前状态汇报给用户

## 文件结构

```
~/todo/
├── inbox.md      # 任务入口
├── active.md     # 执行中
├── backlog.md    # 待处理
└── README.md     # 完整规则
```

## 优先级

- P0: 紧急
- P1: 重要
- P2: 常规
- P3: 低优
- P4: 再说

## 示例

**用户说**："帮我加一个 P1 任务，研究 GPU 板块"

**执行**：
1. 写入 `~/todo/inbox.md`：
```
[P1] 研究 GPU 板块
```
2. 回复："✅ 已添加到 inbox，等 cron 触发或你要求检查时我会处理。"
